## 二、Stat Product — PVP 最重要的單一指標

### 2-1 定義

```
Stat Product = 攻擊力 × 防禦力 × HP（實際值，非基礎值）
```

CSV 欄位「Stat Product」即為此值。同 CP 上限下，**Stat Product 越高 = 存活更久 + 承傷更多**。

### 2-2 為何 Stat Product 比 CP 更重要

PVP 採回合制打傷害，不看暴擊。高 DEF 和高 HP 讓你撐過更多回合。高 ATK 讓你造成更多傷害。Stat Product 是三者的積，在同等 CP 下代表整體 PVP 效能。

### 2-3 各聯盟參考門檻（來自 CSV 資料）

| 聯盟 | CP 上限 | 優秀 Stat Product | 頂級 Stat Product |
|------|--------|------------------|------------------|
| 超級聯盟 | 1500 | ≥ 1,800,000 | ≥ 2,100,000 |
| 高級聯盟 | 2500 | ≥ 3,800,000 | ≥ 4,400,000 |
| 大師聯盟 | 10000 | ≥ 8,000,000 | ≥ 11,000,000 |

> CSV 中評分 90+ 的寶可夢，Stat Product 幾乎都在上表「優秀」以上。

## 三、招式效率分析

### 3-1 快招評估：DPT 和 EPT

快招（一般招式）有兩個關鍵參數：
- **DPT**（Damage Per Turn）：每回合傷害
- **EPT**（Energy Per Turn）：每回合能量回充

```
優先選擇 EPT ≥ 3.0 的快招 → 能快速蓄積特招能量
若 EPT 偏低但 DPT 很高，仍有爆發壓力
```

### 3-2 特招能量消耗（CSV「幾下可用特招」欄）

CSV 中「幾下可用特招1」代表需要幾次快招才能放出第一個特招：

| 幾下可用特招 | 評價 | 戰術用途 |
|------------|------|---------|
| **3–5 下** | ⚡ 高壓型 | 快速蓄招，可頻繁迫使對手開盾 |
| **6–9 下** | ⚖️ 均衡型 | 攻守兼備，壓力與傷害取得平衡 |
| **10–15 下** | 🛡️ 重砲型 | 一次性高傷害，等待關鍵時機放招 |

### 3-3 精英/傳承招式標記

CSV 中招式名稱後標記說明：

| 標記 | 說明 |
|------|------|
| `*` | 傳承招式（舊版 CD/活動限定），需精英快招糖（ET）解鎖 |
| `†` | 當前賽季活動招式或首次登場的限定招式 |

**重要**：帶 `*` 或 `†` 的寶可夢通常排名顯著高於無標記版本，是搜尋最佳陣容時優先考量的關鍵。

### 3-4 招式組合的實戰邏輯

```
快招屬性 → 最好與寶可夢本身屬性相同（STAB 加成 × 1.2）
特招1 → 低能量消耗（3–7 下）作為「誘盾彈」
特招2 → 高傷害 or 覆蓋弱點（作為收場技）

理想組合範例：
  帝王拿波（水/鋼）
  ・金屬音（EPT 高）
  ・加農水砲* （5 下，STAB 誘盾）
  ・啄鑽 （5 下，覆蓋弱點）
```

## 四、暗影寶可夢（Shadow）

### 4-1 暗影加成公式

```
暗影 ATK = 基準 ATK × 1.20（+20%）
暗影 DEF = 基準 DEF × 0.833（-16.7%）
```

### 4-2 使用時機

| 情境 | 暗影 vs 一般 | 理由 |
|------|------------|------|
| 對手剩 1 個護盾 | 暗影 ✅ | 高 ATK 強殺更有效率 |
| 需要撐時間等換場 | 一般 ✅ | 較高 DEF 讓你撐更久 |
| 對手殘血需秒殺 | 暗影 ✅ | 更高傷害可直接終結 |
| 鏡面戰（同種對決）| 暗影 ✅ | 先出高 ATK 傷害佔優 |

### 4-3 CSV 中暗影的評分差異

觀察 CSV 資料，同一隻寶可夢暗影版本通常比一般版高 1.5–3 分（如「七夕青鳥」非暗影 93.2 / 暗影 92.6 — 此案例暗影反而略低，因此需具體看數字而非一概而論）。

## 五、IV 最佳化概念

### 5-1 為何 PVP 不追求 ATK 高 IV

在 CP 上限制下（如 CP1500）：
- **低 ATK IV → 相同 CP 可以提升到更高等級 → 更高的 DEF 和 HP 實際值 → 更高 Stat Product**
- 大多數 CP1500/CP2500 的最佳 IV 是低攻 IV（0 或 1）配高防 HP IV（12–15）

### 5-2 各聯盟推薦 IV 方向

| 聯盟 | ATK IV | DEF IV | HP IV | 原因 |
|------|--------|--------|-------|------|
| 超級 CP1500 | 0–2 | 13–15 | 13–15 | 低 ATK = 更高等級 = 更高 Stat Product |
| 高級 CP2500 | 0–4 | 13–15 | 13–15 | 同上 |
| 大師 CP10000 | 15 | 15 | 15 | 無 CP 上限，直接最高 |
| 各類特殊盃 | 視具體 Pokemon 而定 | 通常低 ATK 較好 | — | |

### 5-3 實用判斷：等級 × IV 組合

```
最佳：Level 20–27 + ATK IV 0–2（榜單寶可夢通常在此範圍）
可用：Level 19–30 + ATK IV 3–7
備用：Level 40 + ATK IV 15（適合大師聯盟）
```

CSV 中「等級」欄顯示榜單中寶可夢的實際等級，可用作 IV 優化參考。

## 六、護盾管理策略

### 6-1 護盾的整體節奏

每位訓練家持有 **2 個護盾**，如何分配攸關勝負：

```
護盾節奏原則：
  開局 → 對手通常用護盾保存主力
  中盤 → 護盾差距形成優勢；比對手多 1 盾是決定性優勢
  殘局 → 無盾方面臨特招直接命中，攻守逆轉
```

### 6-2 誘盾（Shield Baiting）

用低能量消耗的特招先出，讓對手誤以為是高傷害技而開盾，保留高傷害技待對手無盾時出：

```
戰術範例：
  ① 蓄到 5 下放「誘盾技」（如加農水砲）→ 對手開盾
  ② 繼續蓄招
  ③ 對手無盾時放「終結技」（高傷害特招）→ 直接大量扣血
```

### 6-3 何時該開盾

| 情境 | 建議 |
|------|------|
| 主力 HP 充足，可能被秒殺 | 開盾保護主力 |
| 剩最後一隻主力 | 盡量保盾到殘局 |
| 對手特招是誘盾技（低傷） | 不開盾，節省護盾資源 |
| 殘局己方無護盾 | 換出抵抗型寶可夢硬撐 |

## 七、換場（Switch）時機決策

### 7-1 主動換場 vs 被動換場

```
主動換場：主動切換以造成屬性優勢（消耗 60秒 CD）
被動換場：寶可夢倒地被迫換出（無 CD）

換場規則：換場後 60 秒內無法再次主動換場
```

### 7-2 最佳換場時機

| 換場時機 | 說明 |
|---------|------|
| 能量優勢換場 | 蓄滿能量後換出，讓對手換出也清空對方能量積累 |
| 對手殘血不換 | 讓對手補上特招，消耗對手護盾 |
| 自身被雙剋時 | 盡快換出免受 4x 傷害 |
| 對手剛換出 | 此時對手能量剛重置，可換出剋制型寶可夢壓制 |

### 7-3 換場搭配（Sack 戰術）

犧牲一隻較弱的寶可夢快速倒地，強迫對手換場，然後在對手換出的 60 秒 CD 期間換出自己的最強剋制者：

```
Sack 流程：
  ① 我方主力瀕死時不開盾 → 讓主力倒地
  ② 換出「犧牲者」引對手換場
  ③ 對手換場 CD 啟動
  ④ 換出真正的剋制型主力 → 無 CD 限制直接出招
```

## 十、資源分配優先級

### 10-1 星塵（Stardust）優先投入順序

```
① CP1500 的頂榜常駐寶可夢（低等級強化 + 解鎖第二招）
② 傳承招式寶可夢（精英糖解鎖 * 招式）
③ CP2500 的頂榜寶可夢
④ 大師聯盟（投入門檻最高，留到後期）
```

### 10-2 糖果（Candy）優先投入

```
① 強化至 CP 上限等級（省糖：低 ATK IV 寶可夢等級較低，所需糖較少）
② 精英招式（ET）解鎖 * 招式（消耗精英快招糖/精英特攻糖）
③ 開放第二招（消耗星塵 + 糖，星塵更貴，優先考慮主力）
```

### 10-3 開第二招建議（來自 CSV 欄「開第二招星塵花費」）

| 花費（星塵）| 評估 |
|-----------|------|
| 10,000 | 極低門檻，幾乎都值得 |
| 50,000 | 主力才開，確認長期使用再投 |
| 75,000 | 高門檻，只給頂級主力 |
| 100,000 | 傳說/神獸等級，慎重評估 |

## 十二、CSV 欄位對照表（使用工具時的欄位解讀）

|---------|------|----------------|

## 十三、與 V1 的銜接

V1 的屬性分析框架（一至八節）**完整保留**，V2 作為「更深一層的過濾器」：

1. **先用 V1** 找出屬性正確的候選清單
2. **再用 V2** 從 Stat Product、招式效率、暗影加成三個維度挑出最優解
3. **最後套用護盾/換場決策**進行實戰操作

> 兩版本互補，V2 不取代 V1，而是在屬性分析通過後進一步精篩最優陣容。
