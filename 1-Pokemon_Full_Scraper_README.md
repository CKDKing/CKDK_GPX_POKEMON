# pokemon_full_scraper.py

寶可夢圖鑑爬蟲，從台灣官方寶可夢網站擷取 #0001～#1025 的完整資料，**包含所有地區型態與特殊型態**。

---

## 輸出檔案

**`POKELIST.csv`**（UTF-8 with BOM，可直接用 Excel 開啟）

| 欄位 | 說明 | 範例 |
|------|------|------|
| 代號 | 圖鑑編號（不含前置零） | `37` |
| 名稱 | 該型態的名稱 | `六尾` |
| 樣貌 | 型態關鍵字 | `阿羅拉` |
| 屬性 | 屬性，多屬性以 `\|` 分隔 | `草\|毒` |
| 弱點 | 弱點，多弱點以 `\|` 分隔 | `火\|飛行\|冰\|超能力` |

### 樣貌欄位規則

| URL 格式 | 頁面 subname | 樣貌填入 |
|----------|-------------|---------|
| `0037`（無底線） | — | `一般` |
| `0037_1` | `阿羅拉的樣子` | `阿羅拉`（去除「的樣子」）|
| `0386_1` | `攻擊形態` | `攻擊`（去除「形態」）|
| `0647_1` | `覺悟的樣子` | `覺悟` |
| `0150_1` | （無 subname）main-name = `超級超夢Ｘ` | `超級超夢Ｘ` |
| `0479_1` | `加熱洛托姆`（無可去除後綴） | `加熱洛托姆` |

## 運作邏輯

```
對每個代號 #0001 ~ #1025：
  1. 抓取基礎 URL（0001）          → 樣貌 = "一般"
  2. 依序嘗試 0001_1, 0001_2, ...  → 樣貌 = 型態關鍵字
  3. 遇到 HTTP 404 即停止此代號
  4. 結果即時寫入進度暫存檔（斷點續傳用）

全部完成後依「代號 → 一般優先 → 其他型態」排序，輸出 POKELIST.csv
```

### 型態名稱萃取邏輯

```
URL 無底線 (_)
  └─ 樣貌 = "一般"

URL 有底線 (_N)
  ├─ 頁面有 subname 文字
  │    ├─ 結尾含「的樣子」→ 去除後綴
  │    ├─ 結尾含「形態」  → 去除後綴
  │    └─ 其他           → 直接使用原文
  └─ 頁面無 subname（超級進化等）
       └─ 使用 main-name 作為型態標籤
```

## 主要設定參數

位於腳本頂部，可依需求調整：

| 參數 | 預設值 | 說明 |
|------|--------|------|
| `START` / `END` | `1` / `1025` | 抓取的代號範圍 |
| `MAX_WORKERS` | `5` | 並發請求數（勿設過高）|
| `DELAY` | `0.25` 秒 | 每次請求間隔（避免被封鎖）|
| `MAX_RETRIES` | `3` | 請求失敗最多重試次數（指數退避）|
| `MAX_FORMS` | `25` | 每個代號最多嘗試幾個 `_N` |
| `OUTPUT_FILE` | `POKELIST.csv` | 最終輸出檔名 |
| `TEMP_FILE` | `POKELIST_full_progress.csv` | 斷點續傳暫存檔名 |

---

## 斷點續傳

中途中斷不會遺失進度。中間結果即時存於 `POKELIST_full_progress.csv`，重新執行時自動偵測並從上次完成的代號繼續。全部完成後暫存檔自動刪除。

## pandas 分析範例

多值欄位以 `|` 分隔，方便 pandas 展開：

```python
import pandas as pd

df = pd.read_csv("POKELIST.csv")

# 展開屬性進行統計
df["屬性"].str.split("|").explode().value_counts()

# 篩選特定型態
df[df["樣貌"] == "阿羅拉"]
df[df["樣貌"] == "一般"]

# 找所有有冰屬性的寶可夢（含所有型態）
df[df["屬性"].str.contains("冰")]

# 找弱點包含火的所有型態
df[df["弱點"].str.contains("火")]

# 找被四倍傷害（4x 弱點）的組合：弱點 >= 5 種的
df[df["弱點"].str.split("|").apply(len) >= 5][["代號","名稱","樣貌","弱點"]]
```
