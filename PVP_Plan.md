# 寶可夢對戰屬性規劃指南

> **最新更新**：分析系統已整合 CP 評估機制。可於「PVP對戰」側邊欄選擇評估等級（LV15/LV20/LV25/LV40/LV50），結果依「屬性評級 → 同等級內 CP 高低」排序，CP 過低（<500）會顯示 ⚠️ 警示。

> 適用場景：已知對手單一寶可夢的屬性，規劃 1v1 或 3v3 出場順序

---

## 一、核心概念

### 傷害倍率

| 狀況 | 倍率 | 說明 |
|------|------|------|
| 超級有效（剋制） | × 2 | 攻擊屬性剋制對方屬性 |
| 效果不大 | × 0.5 | 對方屬性抵抗此攻擊 |
| 無效（免疫） | × 0 | 完全免疫 |
| **雙剋（雙屬性皆剋）** | **× 4** | 攻擊屬性同時剋制對方兩種屬性 |
| 相消（2x × 0.5x） | × 1 | 一剋一抗，效果歸零 |

### 決策優先級

```
[最高] 排除 ── 我方屬性被對方雙剋（受到 4x 傷害）
[次高] 選入 ── 我方攻擊能雙剋對方（造成 4x 傷害）
[中]   排除 ── 我方屬性被對方單剋（受到 2x 傷害）
[中]   選入 ── 我方攻擊能單剋對方（造成 2x 傷害，且不相消）
[低]   中立 ── 既無剋制也無弱點，作為保底選擇
```

## 二、分析步驟（通用框架）

### Step 1｜確認對手屬性

從圖鑑取得對手寶可夢的屬性（1 或 2 種）

```
範例：對手 妙蛙種子 → 屬性 草 ｜ 毒
```

---

### Step 2｜找出「能雙剋對方」的攻擊屬性

**規則：攻擊屬性必須同時出現在對方每一種屬性的弱點清單中**

```
對手單屬性 A：
  → 直接查 A 的弱點欄 → 全部皆可「剋制」，無所謂雙剋

對手雙屬性 A ｜ B：
  → 查 A 的弱點 ∩ 查 B 的弱點 → 交集 = 雙剋屬性（4x）
  → A 弱點 ∪ B 弱點（排除交集）= 單剋屬性（需確認不相消，見 Step 2b）
```

**Step 2b｜確認單剋是否相消**

當攻擊屬性只剋制對方一種屬性時，需確認對方另一屬性是否「抵抗」此攻擊：

```
攻擊效果 = (vs 屬性A 倍率) × (vs 屬性B 倍率)
  2x × 1x = 2x ✓ 有效單剋
  2x × 0.5x = 1x ✗ 相消，不值得用
  2x × 0x   = 0x ✗ 被免疫
```

---

### Step 3｜找出「被對方雙剋」的我方屬性

**規則：對方每一種屬性的攻擊都能超有效命中我方**

```
對手雙屬性 A ｜ B：
  → 查「A 型攻擊打哪些屬性 2x」的清單
  → 查「B 型攻擊打哪些屬性 2x」的清單
  → 兩清單的交集 = 我方最應排除的屬性（被雙剋）
  → 其餘出現在任一清單的屬性 = 被單剋（避免優先帶入場）
```

## 三、1 v 1 決策流程

```
已知對手屬性
      │
      ▼
  有雙剋攻擊？─── 是 ──→ 選此攻擊屬性的寶可夢
      │                   且確認自身不被對手雙剋/單剋
      │ 否
      ▼
  找有效單剋（≥2x 不相消）
      │
      ▼
  從候選中排除「被對手剋制」的屬性
      │
      ▼
  剩餘候選中選攻防最均衡者
```

**選角評分標準（同分優先上方）**

| 優先級 | 我的攻擊效果 | 我的防守狀態 |
|--------|------------|------------|
| S（最優） | 雙剋 4x | 不被剋 |
| A | 單剋 2x | 不被剋 |
| B | 雙剋 4x | 被單剋 2x（攻強但脆）|
| C | 單剋 2x | 被單剋 2x（平手邊緣）|
| D | 無剋制 | 不被剋（純保底）|
| 排除 | 任何 | 被雙剋 4x |

## 四、3 v 3 組隊策略

### 4-1 核心判斷原則

> **關鍵觀念**：3v3 不一定每隻都要上場迎戰所有對手。
> 我方某隻寶可夢的「單一屬性被某對手單剋（×2）」是可以接受的，
> 因為你可以選擇不讓它對上那隻對手。
> 只有被任一對手「雙剋（×4）」才是真正危險，必須排除。

---

### 4-2 分析邏輯（系統自動執行）

交叉比對 3 位對手的弱點屬性，找出能同時剋制最多對手的候選寶可夢，依下表 12 個層次評分排序。

**術語說明**
- **有優勢**：候選寶可夢的屬性能對該對手造成 ×2 以上傷害
- **雙剋**：攻擊屬性同時剋制對手的兩種屬性（×4 傷害）
- **被單剋**：對手某一屬性能對我方造成 ×2 傷害（可接受，不強制迴避）
- **被雙剋**：對手兩種屬性都能打我方 ×2（=×4，絕對排除）

**評分表（同評級內以 LV40 CP 由高到低排序）**

| 評級 | 層次 | 條件 | 備註 |
|------|------|------|------|
| **S** | T1 | 對 3 隻全有優勢 **＋ 含雙剋** | 最優先 |
| **A** | T2 | 對 3 隻全有優勢（無雙剋，無被單剋） | |
| **A** | T3 | 對 3 隻全有優勢（1 隻交叉比對被單剋，沒關係） | |
| **A** | T4 | 對 3 隻全有優勢（2 隻交叉比對被單剋，沒關係） | |
| **B** | T5 | 對 2 隻以上有優勢 **＋ 含雙剋** | |
| **B** | T6 | 對 2 隻以上有優勢（無雙剋，無被單剋） | |
| **B** | T7 | 對 2 隻以上有優勢（1 隻交叉比對被單剋，沒關係） | |
| **C** | T8 | 對 2 隻以上有優勢（2 隻交叉比對被單剋，沒關係） | |
| **C** | T9 | 對 1 隻有優勢 **＋ 含雙剋** | |
| **C** | T10 | 對 1 隻有優勢（無雙剋，無被單剋） | |
| **D** | T11 | 對 1 隻有優勢（交叉比對被單剋，沒關係） | |
| **D** | T12 | 對任一對手均無剋制優勢 | 保底中立 |
| **排除** | — | 被任一對手「雙剋（×4）」 | 絕對避免 |

---

### 4-3 CP 篩選原則

```
屬性剋制決定「能否上場」，CP 決定「上場有沒有效果」。
同評級內以 LV40 CP 由高到低排序。

⛔ CP < 500    → 即使屬性正確也難以取勝
⚠️  CP 500–999 → 偏低，需操作優勢才有機會
✅ CP 1000+   → 可用範圍
⭐ CP 2000+   → 優先考慮
```

---

### 4-4 結果介面說明

分析結果包含：

1. **對手隊伍卡片** — 顯示 3 隻對手的屬性、弱點、ATK/DEF/HP、CP
2. **CP 過濾拉桿** — 以 LV40 為基準，左右拖動篩選 CP 區間
3. **評級分組** — S / A / B / C / D / 排除，每張卡顯示「vs1 / vs2 / vs3」個別評級
4. **per-opponent 標籤** — 每張候選卡的 vs 標籤顯示對每位對手的個別評級，方便研判上場組合

---

### 4-5 隊伍組建建議

```
Step A — 主力（Ace）
  → 選 S 或 A 評級，對最多對手有效，CP 盡可能高

Step B — 補位（Sub）
  → 選與主力攻擊屬性不同的 A / B 評級
  → 補足主力無法覆蓋的對手弱點

Step C — 保底（Safe）
  → 選 B / C 評級，不被任一對手雙剋
  → 作為前兩隻倒下後的持久備援
```

### 4-6 隊伍健康度檢核

```
□ 三隻中沒有任何一隻被任一對手「雙剋」（4x）
□ 不是三隻都弱於對手同一攻擊屬性
□ 至少一隻能同時剋制 2 隻以上對手（S 或 A 級）
□ 三隻 LV40 CP 均達 1000 以上（低於此值需謹慎）
```

## 六、實戰範例

### 範例 A｜對手 草 ｜ 毒（妙蛙種子）

#### Step 2：找雙剋攻擊
```
草 的弱點：火、冰、毒、飛行、蟲
毒 的弱點：地面、超能力
交集     ：【無】→ 對此組合不存在雙剋攻擊
```

#### 找有效單剋（不相消）

| 攻擊屬性 | vs 草 | vs 毒 | 合計 | 可用？ |
|---------|------|------|------|-------|
| 火 | 2x | 1x | **2x** | ✅ |
| 冰 | 2x | 1x | **2x** | ✅ |
| 飛行 | 2x | 1x | **2x** | ✅ |
| 地面 | 1x | 2x | **2x** | ✅ |
| 超能力 | 1x | 2x | **2x** | ✅ |
| 蟲 | 2x | 0.5x | **1x** | ❌ 相消 |

#### Step 3：找被對方雙剋的屬性
```
草型攻擊效果佳（打2x）：水、地面、岩石
毒型攻擊效果佳（打2x）：草、妖精
交集：【無】→ 不存在被雙剋的屬性

被草單剋（避免）：水、地面、岩石
被毒單剋（避免）：草、妖精
```

#### 3v3 組隊建議

| 位置 | 推薦攻擊屬性 | 避免帶入屬性 | 理由 |
|------|-----------|-----------|------|
| 主力 | 火 or 飛行 | 水、地面、岩石 | 剋草且抵抗草系攻擊 |
| 補位 | 超能力 or 地面 | 草、妖精 | 剋毒，攻擊角度不同 |
| 保底 | 鋼 or 電 | — | 不弱於草/毒，中性抗打 |

## 八、CP 評估補充

> **核心原則**：屬性剋制決定「能不能用」，CP 決定「用起來有沒有效果」。

### 8-1 CP 等級對照

| 來源 | 等級 | 說明 |
|------|------|------|
| 研究任務獎勵 | **LV15** | 最低常見等級 |
| 巢穴 / 孵化蛋 | **LV20** | 最常見入手等級（預設評估）|
| 天氣加成遭遇 | **LV25** | 特殊條件下的增強版本 |
| 最高強化（一般）| **LV40** | 一般玩家滿等 |
| 最高強化（XL糖）| **LV50** | 需要特殊資源 |

### 8-2 CP 警示標準

| CP 範圍 | 評估 | 系統顯示 |
|---------|------|----------|
| < 500 | ⛔ 過低，即使有剋制優勢也極難勝出 | 紅色 ⚠️ |
| 500 – 999 | ⚠️ 偏低，需搭配高技能操作才有機會 | 黃色 |
| 1000 – 1999 | ✅ 可用範圍，屬性剋制能發揮作用 | 綠色 |
| ≥ 2000 | ⭐ 高戰力，優先考慮 | 綠色 |

### 8-3 分析系統使用方式

```
1. 在「PVP對戰」側邊欄選擇評估等級
   （LV15 / LV20 / LV25 / LV40 / LV50）

2. 輸入對手寶可夢並點選「開始分析」

3. 結果排序邏輯：
   第一優先 → 屬性評級（S > A > B > C > D > 避免）
   同評級內 → CP 由高到低排列

4. CP 顏色指示：
   綠色（≥1000）→ 直接可用
   黃色（500–999）→ 謹慎評估
   紅色（<500）+ ⚠️ → 不建議使用
```
